Progress Quest Slog: Chinbreak Island

Welcome to PQS:CI. You can skip the rigamore and play the game, or read on for the rigamarole, including links to many moving parts.

What it is

There are many layers to this onion.

Chinbreak Island is a game in the Progress Quest Automatic Universe™, with appearance, style and gameplay similar to the original Progress Quest released in 2002. It's an idle/zero-player game, meaning your character does the thinking, clicking and hotkey-pressing for you, while you either watch or occupy yourself with other more important tasks.

Strats

However, unlike the original, where your character's play style is fixed, here your character's actions are determined by a programmable strategy that it follows. A player selects a strategy at the time of character creation. It may be built-in to the game, or gotten from elsewhere.

But! However! You may also write or modify a character strategy yourself.

Slog VM

The active strategy is followed in the game by operating a virtual machine within which the strategy 1) observes the state of the game and 2) gives instructions to the character, analogous to a human (or humanoid) player watching a game and clicking buttons and so on.

This virtual machine is called the Slog Virtual Machine.

Tools

There is an assembly language and an associated macro assembler which targets the SVM, but, more practically, there is a higher level language called Slog, and a compliler for it. Those who wish to develop strategies can use these tools, or any other means they may devise, to produce machine code suitable for the SVM.

These tools are written in Node.js and can be used from the command line by checking out the code.

IDE

There is also an in-browser integrated development environment of sorts, which can be useful for trying, tweaking and debugging strategies.

This is Alpha

The compiler and virtual machine seem fairly stable, but the Chinbreak Island module is very much in an alpha stage of development. For example, the play balance is screwy, there's no save/load functionality, and there's probably many, many bugs.

If you have a suggestion or find a bug in the game, IDE, documentation, compiler, assembler, virtual machine, or anything else, please feel encouraged to report it on github.

Future Directions

The first priority is to fix bugs and fix the play balance. After these things there are many follow on ideas to implement, such as

  • multiplayer / PvP / battle royale
  • leader boards
  • other modules, e.g. a dungeon crawler
  • sequels where winning characters can continue on
  • entirely different genres and game styles
